# 这是个好事儿啊

一个可运行的 Codex skill：把倒霉事、破防事、丢脸事，改写成一种粗粝、反鸡汤、拆穿式的安慰回应。

这不是温柔陪伴型 skill。它的核心不是哄，而是先把用户脑子里那套“我完了”“天塌了”“这就是纯坏事”的叙事掰歪，再用更俗、更硬、更不客气的逻辑把事情打回现实。

## 适用场景

- 被拒绝、面试挂了、计划失败
- 感情受挫、分手、关系错配
- 丢脸、破防、比较心、下头
- 想听“拆穿式”“骂醒式”安慰，而不是温柔抚摸

## 不适用场景

- 自伤、自杀、暴力、急症
- 丧亲、重病、真实创伤现场
- 用户明确只要冷分析，不想要风格化回应

## 这个 skill 做什么

触发后，回答会尽量做到这几件事：

1. 第一拍先改判，不顺着“完了”往下演
2. 把大情绪打回具体事实
3. 优先拆脸面、比较心、沉没成本、幻想
4. 用常识、追问和生活化比喻把戏台子掀了
5. 最后只给一个很硬的方向，不写成心理咨询稿

## 风格来源

这个 skill 不是凭空猜口气写的，而是基于：

- 公开访谈与媒体转引
- 本地视频清洗转写
- 研究笔记里的表达 DNA、片段摘录和跨视频共性

当前公开仓库里已经包含：

- skill 本体
- 调研与蒸馏笔记

## 示例

用户输入：

```text
我对象考编上岸了，我没上。我感觉我在这个城市待不下去了，只能回老家了。
```

期望的方向不是：

- 温柔共情三段
- 先讲“你值得被爱”
- 写成标准心理辅导

而是更接近这种路数：

- 先问：`怎么就待不下去了`
- 再拆：`你这是没考上，不是没饭吃`
- 再拆面子：`她上岸了，不等于你成下等人了，是你自己先跪了`
- 最后给一个很硬的现实方向

## 研究边界

这个仓库尽量区分了两类东西：

1. 清洗文本里反复出现、能直接支撑的表达习惯
2. 基于这些习惯推出来的高相似写法

所以它追求的是：

- 可运行
- 风格接近
- 蒸馏过程可检查

不是：

- 冒充真人
- 逐字逐句复刻
- 把所有近似表达都说成原句实录
